<pvcspromotegroup>

Promotes versioned files to the next promotion group.

This task uses the promotegroup PCLI command to promote versioned files.

Parameters

Attribute Type Description Required
acrossbranches bool Gets or sets a value indicating whether the promotion may occur across branches. True
promotiongroup string Gets or sets the promotion group to be promoted. True
entity string Gets or sets the entity involved in the operation. True

password string Gets or sets the password to use when connecting to the project database. False

timeout int The maximum amount of time the application is allowed to execute, expressed in milliseconds. Defaults to no time-out. False

userid string Gets or sets the user ID to use when connecting to the project database. False
verbose bool Determines whether the task should report detailed build log messages. The default is false. False
workspace string Gets or sets the workspace to use when connecting to the project database. False

When passed to an external application, the argument will be quoted when appropriate. This does not apply to the line parameter, which is always passed as is.

<path>

Sets a single command-line argument and treats it like a PATH - ensures the right separator for the local platform is used.

Examples

  • A single command-line argument containing a space character. 

    <arg value="-l -a" />

  • Two separate command-line arguments. 

    <arg line="-l -a" />

  • A single command-line argument with the value \dir;\dir2;\dir3 on DOS-based systems and /dir:/dir2:/dir3 on Unix-like systems. 

    <arg path="/dir;/dir2:\dir3" />

</arg>

Examples

  • Promotes all files in the root of the project database specified by the project-database property. The files are promoted from the DEV promotion group to the next. Promotion will not take place across branches. 

    <pvcspromotegroup projectdatabase="${project-database}" promotiongroup="DEV" entity="/" acrossbranches="false"/>

  • Promotes all files in the project database specified by the project-database property. The files are promoted from the SYSTEST promotion group to the next. Promotion will take place across branches. 

    <pvcspromotegroup projectdatabase="${project-database}" promotiongroup="SYSTEST" entity="/" includesubprojects="true" acrossbranches="true"/>
